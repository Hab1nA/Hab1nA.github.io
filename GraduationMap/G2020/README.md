# 🗺️ 天府七中 G2020 级蹭饭图

> 展示 2020 届高中毕业生大学地理分布的可视化工具。以天地图卫星影像为底图，将全年级四个班同学的大学去向以彩色标记呈现在全国地图上——散落各地的求学轨迹，一张图尽收眼底。

---

## ✨ 功能特性

### 🔒 身份验证
- 页面加载后首先展示验证遮罩，需输入正确的**学校名称**才能解锁地图。
- 口令通过浏览器端 **SHA-256** 哈希比对验证，源代码中不储存任何明文口令，仅保留哈希值。
- 验证通过后导航栏与筛选面板以动画滑入，口令遮罩淡出消失。
- 输入错误时输入框抖动提示，错误信息显示。

### 📚 班级筛选
- 右下角**「班级筛选」面板**提供 1–4 班的复选框，可单独或任意组合勾选。
- 每个班级以独立颜色标识：1 班紫色 `#a74bb6`、2 班绿色 `#10b981`、3 班琥珀色 `#f59e0b`、4 班红色 `#a3292b`。
- 若多个班级有同学就读同一所大学，该标记会**自动合并**为灰色 `#9ca3af`，点击后在信息窗内按班级分组列出所有同学。
- 面板底部实时显示当前已勾选班级的**总人数**。

### 📍 大学地理编码定位
每个大学的坐标通过以下**优先级链条**确定：

1. **预置经纬度优先**——若数据文件中某同学提供了有效的 `latitude` / `longitude` 字段（十进制度，纬度 −90~90、经度 −180~180），则直接以此坐标落点，无需额外网络请求。
2. **坐标冲突检测**——同一大学若不同同学提供了不一致的坐标，系统判定为冲突，自动回退到地理编码搜索。
3. **天地图地理编码**——以上情况均不满足时，通过天地图地理编码服务将"城市名 + 大学名"转换为经纬度坐标。编码请求以 **250ms** 间隔逐条发送，避免触发 API 限流。

### 🔍 搜索功能
顶部搜索框支持**两种检索模式**，按优先级依次匹配：

1. **同学姓名搜索**——在全年级数据中模糊匹配姓名（不区分大小写）。优先精确匹配，命中后地图自动飞行至该大学并打开信息窗。若该生所在班级未被当前筛选勾选，会临时创建标记，同步合并显示所有班级数据。
2. **大学/城市搜索**——若姓名未命中，按大学名称模糊匹配。同样优先精确全名匹配，支持城市名称检索。

当存在**多个搜索结果**时，顶部出现导航条（← 上一个 / 下一个 →），可逐一浏览。关闭搜索（点击 ✕ 按钮、清空输入框、或切换班级筛选）后，临时标记自动清除，原有标记恢复原状。

### 📊 数据统计
点击顶部**「数据统计」按钮**打开统计面板，内容包括：

- **年级总览**——全年级总人数及各班级人数徽标（含缺失数据同学）。
- **城市分布 Top 8**——以水平柱状图展示同学去向最多的城市排名。
- **热门院校 Top 8**——以水平柱状图展示录取人数最多的院校排名。院校名称智能合并——去除括号内校区后缀（如"某某大学（北京校区）"），同时保留**独立办学实体**的完整名称（如"中国地质大学（武汉）"与"中国地质大学（北京）"分别统计，中国石油大学（北京/华东）、中国矿业大学及其北京校区同理）。

### ⚠️ 数据缺失面板
- 部分同学可能缺少大学去向信息。勾选班级后，若存在缺失数据，右下角会在班级筛选面板上方显示**金色边框的「数据缺失」按钮**。
- 点击按钮打开模态框，按班级分组列出缺失信息的具体姓名名单。
- 所有已勾选班级均无缺失数据时，该按钮自动隐藏。

### 🛡️ 容错机制

| 机制 | 说明 |
|------|------|
| **API 额度双通道检测** | 通道一：监听天地图瓦片 `<img>` 加载失败事件（10 秒窗口内 ≥8 次失败判定额度已满）；通道二：监听地理编码连续失败次数（≥5 次触发严重故障告警）。二者任一触发即清空请求队列并提示用户。 |
| **超时保护** | 每条地理编码请求设置 8 秒超时，超时后不计入缓存，允许后续重试。 |
| **版本号隔离** | 切换班级筛选（`markerRenderVersion`）、导航搜索结果（`searchNavRequestId`）、强制清空地理编码队列（`geocodeCancelledVersion`）时均递增对应版本号，使飞行中的异步回调自动判定自身过期并丢弃，避免过期数据污染当前视图。 |
| **地图 API 延迟加载** | `index.html` 末尾的三方 API 引用与 30 次轮询重试（100ms 间隔），确保即使 CDN 延迟也能正确初始化。同时检测占位 TK 并输出控制台错误提示。 |

### 为何选择天地图？

- 卫星影像对中国境内有更高的分辨率覆盖。
- 地理编码对中文大学名称的识别精度高。
- 免费额度足以满足班级内部使用。

## 🧠 核心机制详解

### 地理编码请求队列

```
数据加载 → 按大学分组 → 检查缓存/预置坐标 → 加入队列
                                                    ↓
                                    processNextGeocode() 逐条执行
                                                    ↓
                      天地图 Geocoder 搜索 "城市名 + 大学名"
                                                    ↓
                         成功 → 缓存结果 → 回调添加标记
                         失败 → 缓存 null  → 回调跳过
                         超时 → 不缓存    → 回调跳过 + 错误计数
                                                    ↓
                              间隔 250ms → 处理下一条
```

- **缓存策略**：`geoCache` 对象以大学全名为键，缓存 `T.LngLat` 或 `null`。同一大学只需一次地理编码请求，成功和"未找到结果"均缓存，超时不缓存以便重试。
- **限速保护**：每条请求间隔 250ms，有效避免触发天地图 API 频率限制。
- **防止重复回调**：每条请求使用 `completed` 标志位防止超时回调与正常回调双重触发。

### 多班同校合并算法

1. 遍历所有勾选班级中每个同学的 `university` 字段。
2. 按大学名分组，统计 `totalStudents`、`classNums[]`、`studentsByClass{}`。
3. 若 `classNums.length > 1` → 标记颜色设为灰色 `#9ca3af`，信息窗内仍按班级分节显示。
4. 若仅一个班级 → 使用该班专属颜色。

### 搜索临时标记管理

| 场景 | 处理方式 |
|------|----------|
| 搜索结果中的大学已有现有标记 + 班级全被勾选覆盖 | 直接打开现有标记的信息窗 |
| 搜索结果包含未勾选班级 | 临时替换已有标记的信息窗为"合并全量数据版"，图标按需变灰，关闭搜索时恢复 |
| 搜索结果中的大学无现有标记 | 创建临时标记（`searchPinnedMarker`），关闭搜索时移除 |

## 🚀 部署与配置

### 1. 获取天地图密钥（TK）

访问 [天地图开放平台](https://lbs.tianditu.gov.cn/) 注册并申请 JavaScript API 密钥。

### 2. 替换密钥

编辑 `index.html`，将两处 `tk=7403bbf65cbf354d6a9e3574134f9789` 替换为你的 TK：

- **第 296 行**：天地图 API 脚本引用中的 `tk` 参数。
- **第 325 行**（`js/main.js` 中地图初始化函数 `onTMapCallback` 里）：卫星影像 WMTS 瓦片 URL 中的 `tk` 参数。

> ⚠️ **重要**：必须同时替换这两处，否则地图瓦片或地理编码服务将不可用。

### 3. 管理口令

1. 使用在线 SHA-256 工具或浏览器控制台计算期望口令的哈希值：
   ```javascript
   // 在浏览器控制台运行
   crypto.subtle.digest('SHA-256', new TextEncoder().encode('你的口令'))
     .then(buf => Array.from(new Uint8Array(buf)).map(b => b.toString(16).padStart(2, '0')).join(''))
     .then(console.log);
   ```
2. 将得到的 64 位十六进制哈希值添加到 `js/main.js` 的 `validHashes` 数组中（第 196 行附近）。
3. 可同时存放多个合法口令的哈希值。

### 4. 更新班级数据

编辑 `data/class1.js` ~ `data/class4.js`，按上述数据格式更新同学信息。文件修改后刷新页面即可生效。

### 5. 部署

整个目录为纯静态资源，可直接部署到任意静态文件服务器（GitHub Pages、Nginx、Apache、Vercel、Netlify 等）。无需构建步骤，无需后端服务。

```
# 本地快速预览（Python 3）
cd GraduationMap/G2020
python -m http.server 8080
# 浏览器打开 http://localhost:8080
```

## 🛠️ 开发说明

### 代码风格

- JavaScript 采用 **ES5 兼容风格**（`var` 声明、`function` 关键字），确保在旧版浏览器中也能正常运行。
- 不使用 Babel、Webpack 等构建工具，所有脚本通过 `<script>` 标签直接加载。
- 全局变量通过命名约定避免冲突（`ALL_CLASS_DATA`、`geoCache`、`activeMarkers` 等）。

### 全局变量清单

| 变量 | 用途 |
|------|------|
| `map` | 天地图 Map 实例 |
| `geocoder` | 天地图 Geocoder 实例 |
| `geoCache` | 地理编码结果缓存 `{ "大学名": T.LngLat \| null }` |
| `activeMarkers` | 当前地图上的所有标记数组 |
| `activeMarkerByUniversity` | 大学名 → Marker 索引 |

### 添加新班级

如需添加第 5 个班级：

6. 在 `js/main.js` 中：修改 `CLASS_COLORS` 添加第 5 班颜色；在 `DOMContentLoaded` 初始化中将循环范围从 4 扩展到 5；在统计函数中将遍历范围从 `1..4` 扩展到 `1..5`。
